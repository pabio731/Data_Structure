



	자료구조의 필요성
	자료들을 효율적으로 저장하여 메모리를 아끼고 프로그램 수행시간을 최소화 할 수 있다.
	또한 효율적인 문제 해결 방법으로 알고리즘을 배우기 위해 자료구조가 꼭 선행되어야 한다.

	
	저장하는 자료의 분류
1. 단순 구조.	:정수, 실수, 문자, 문자열
2. 선형 구조	:리스트, 스택, 큐, 덱
3. 비선형 구조.	:트리, 그래프
4. 파일 구조

1. 단순 구조
int, float, double등 일반적인 타입들을 말함.

2. 선형 구조
각각 자료들 앞뒤와의 사이 관계가 일대일(1:1)
NULL - 1번자료[0] - 2번자료[1] - 3번자료[2] - NULL 와 같은 형태

3. 비선형 구조
자료간의 관계가 계층 구조 또는 망(Network) 구조

4. 파일 구조
보조 기억 장치에 저장되는 파일의 자료 구조를 말함. 디스크에 저장되는 것을 가정으로 하며
주로 메모리에 한번에 올릴 수 없는 대용량의 자료들을 대상으로 한다
순차적 구조, 상대적 파일 구조, 색인 구조, 다중 키 파일 구조가 있다



추상 자료형
자료 구조를 표현하는 방법 중의 하나. 정보 은닉을 위해서 사용하며 궁극적으로는 개발 효율성이 올라간다(불필요한것 제거)
모든것을 알지 못하더라도 활용 가능
구현 로직은 몰라도 인터페이스를 통해서 활용 가능하게 하는 것이 목표

자료와 자료
자료형 = 자료 + 연산
ex) 정수 자료형-> 1,2,3,4.... + 사칙연산

즉 추상 자료형이란 자료형을 이용해서 필요한 부분만 보여주고 불필요한 부분을 감춰두는 것
